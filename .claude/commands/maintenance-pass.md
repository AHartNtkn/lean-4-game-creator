# /maintenance-pass

Use when doing technical or pedagogical maintenance on an existing repo.

## Prompt text

Run a maintenance pass on this lean4game repo.

First infer:
- house style,
- current coverage,
- current granularity,
- and current learner-facing pain points.

Then perform or plan the maintenance in a way that preserves course quality.

Check all of:
- toolchain / dependency drift,
- local-run and publish path,
- dependency drift,
- missing docs,
- missing aliases,
- overbroad or stale worlds,
- pset quality,
- boss integrity,
- and issue-driven pedagogy repairs.

Return exactly these sections:

1. Maintenance mode
2. Observed problems
3. Coverage / granularity impact
4. Files to edit
5. Patch plan
6. House-style constraints
7. Regression risks
8. Release notes phrased in learner-facing terms
9. Post-patch playtest plan
