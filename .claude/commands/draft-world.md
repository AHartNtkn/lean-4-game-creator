# /draft-world

Use when authoring one world.

## Prompt text

Design or write a lean4game world as a coherent chapter.

First decide:
- the world type,
- the world promise,
- the world’s center of gravity,
- and the coverage items it must close.

Then produce a level ladder in which each level has:
- a dominant lesson,
- a novelty budget,
- inventory behavior,
- hint philosophy,
- and a conclusion role.

The world must have an explicit boss design if it is a lecture or integration world.

Do not let the level sequence become a bag of theorem statements.
I care about proof-move coverage, transfer, and granularity.

Return exactly these sections:

1. World promise
2. World type and rationale
3. Coverage closure table
4. Level ladder
5. World intro draft
6. Inventory plan
7. Boss design
8. Files to create or edit
9. Risks and likely learner failure points
10. Playtest focus

If I asked for code, append the full code after the design section.
