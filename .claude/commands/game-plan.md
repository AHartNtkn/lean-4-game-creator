# /game-plan

Use when starting a new course or redesigning an existing one.

## Prompt text

Design a lean4game course plan that is optimized for course quality, not just technical validity.

Important:
- coverage and granularity are mandatory;
- theorem coverage alone is not enough;
- I care about teaching the how of proof, not only the what of theorems.

Build:
1. a content map,
2. a proof-move map,
3. a Lean / notation map,
4. a misconception map,
5. a coverage plan,
6. a world graph,
7. an inventory release plan,
8. a boss map,
9. a transfer / retrieval plan,
10. and a risk report.

Use a deliberate mix of world archetypes such as onboarding, lecture, pset, and review where appropriate.

Return exactly these sections:

1. Course promise
2. Learner profile
3. Coverage map
4. Granularity plan
5. World graph
6. Inventory release plan
7. Boss map
8. Transfer and retrieval plan
9. Misconception plan
10. Major risks
11. Recommended first three worlds to author
